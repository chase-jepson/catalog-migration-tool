## CRITICAL — Fix Immediately

### 1. `formatDecimal2` corrupts multiples of 10
**File:** `lib/inventory-transformer.ts:99`

```ts
return fixed.replace(/\.?0+$/, "") || "0";
```
`10.00` → strips `.00` then trailing `0` → `"1"`. Every inventory quantity that's a multiple of 10 is silently truncated. `100` → `"1"`, `200` → `"2"`.

**Fix:** `return fixed.replace(/(\.\d*?)0+$/, "$1").replace(/\.$/, "");`

### 2. `dedupKey` in `processReceipts` still uses `|` separator
**File:** `lib/inventory-transformer.ts:253`

The `sumByGroup` key collision was fixed to `\0`, but `dedupKey` still uses `|`:
```ts
const dedupKey = (r) => `${r.ExternalPackageId}|${r.VendorName}|${r.ReceiveDate}|${r.OrderTitle}`;
```
Pipe-separated vendor names (confirmed in production data) collide and silently drop receipt rows.

**Fix:** Replace `|` with `\0` to match the pattern established in `sumByGroup`.

### 3. Refreshed OAuth tokens never persisted
**File:** `entrypoints/background/auth.ts:134-140`

`refreshAccessToken` returns new tokens but they're never written back. Next call reads the old expired token, triggers another refresh. If refresh tokens are single-use, the second refresh fails → user locked out mid-migration.

**Fix:** After successful refresh, write tokens back to page localStorage via `chrome.scripting.executeScript`.

### 4. State persistence saves but never restores
**File:** `components/wizard/WizardShell.tsx:159-166`

`clearMigrationState()` is called unconditionally on mount. `loadMigrationState()` is imported but never called. Users lose all progress when the drawer closes accidentally.

**Fix:** Load state on mount, prompt "Resume previous migration?", then clear only on user choice.

## HIGH PRIORITY — Data Accuracy (the 80% → 99% gap)

These are the findings most likely contributing to the 20% transformation error rate.

### 5. `priceToCents` silently converts missing/zero prices to $0.01
**File:** `lib/transformer.ts:321-329`, `lib/validator.ts:116`

Missing prices become `"1"` (1 cent). Validator blocks `"0"` but allows `"1"`. Products import at $0.01 with no warning.

**Fix:** Return `"0"` for missing prices. Let the validator surface it as an error so the user can fix it in the review step.

### 6. `extraContext` joins ALL columns for category resolution
**File:** `lib/transformer.ts:531`

```ts
const extraContext = Object.values(row).join(" ");
```
A vendor named "Flower Shop" triggers a "Flower" category match. An address with "Mint St" triggers "Mints". This is a major source of miscategorization.

**Fix:** Scope `extraContext` to relevant columns only: `productName`, `category`, `subCategory`, `description`, `strain`. Exclude vendor, address, notes, IDs.

### 7. `sugar` keyword maps edibles to Extract
**File:** `lib/category-mapper.ts:83`

`\b(sugar)\b` matches "Brown Sugar Cookie" → Extract instead of Edible. The Extract rule fires before Edible rules when matching against combined text.

**Fix:** Add exclusion context — if "cookie", "candy", "gummy", or other edible keywords are also present, skip the Extract match. Or require `sugar` to appear alongside concentrate-specific context ("sugar wax", "live sugar").

### 8. `deriveStatus` only handles Dutchie's "Is retired" format
**File:** `lib/transformer.ts:160-162`

Only `"yes"` → `"inactive"`. Cova's "Disabled", Blaze's "Archived", Flowhub's status values all silently become "active". Retired/disabled products import as active and appear on menus.

**Fix:** Expand to handle common status values across all POS systems: `inactive/disabled/archived/retired/discontinued/no → "inactive"`.

### 9. Blaze maps THC and weight to the same column
**File:** `lib/constants.ts:234`

Both `weight` and `thc` map to `"Custom Weight Measurement"`. For non-flower products, the weight value is incorrectly parsed as THC content.

**Fix:** Only map `weight` to this column. THC for Blaze should come from a THC-specific column or be left empty for name-based extraction.

### 10. Variant dedup silently drops rows with duplicate SKUs
**File:** `lib/csv-generator.ts:203-207`

Variants deduped by `skuBarcode` — first wins, no warning. Blaze commonly has duplicate Product IDs across weight tiers. User never sees what was dropped.

**Fix:** Add duplicate `skuBarcode` detection to the validator as a warning. Show which rows were deduped in the review step.

### 11. Vendor join is case-sensitive — no normalization
**File:** `lib/inventory-transformer.ts:521-537`

"ABC Farms" ≠ "ABC farms". POS exports frequently have casing inconsistencies in vendor names. Unmatched vendors lose all vendor metadata.

**Fix:** Normalize join keys to lowercase (`.toLowerCase()`) before comparison.

### 12. `extractGramsFromName` misses fractional ounces
**File:** `lib/transformer.ts:254-256`

"Quarter OZ" and "2oz" don't extract correctly. Only `\bounce\b` matches (returns 28g). "Half oz" is handled in category-mapper but not here.

**Fix:** Add patterns for `quarter\s*oz` → 7, `half\s*oz` → 14, `(\d+)\s*oz` → n*28, `eighth` → 3.5.

### 13. Meadow/IndicaOnline map strain and classification to same column
**File:** `lib/constants.ts:262-263, 275-276`

Strain name (e.g., "Blue Dream") fed into `normalizeClassification` → returns empty. The mapped classification column is effectively unused. Meadow also has a leading space in the header name.

**Fix:** For these POS systems, only map the column to `strain`. Classification should fall through to name-based detection, which already works.

### 14. No duplicate `productId` / `skuBarcode` validation
**File:** `lib/validator.ts`

Validator checks field validity but never uniqueness. Duplicates are silently resolved (first wins) in CSV generation. User has no chance to review.

**Fix:** Add cross-row uniqueness validation as warnings in `validateDerivedRows`.

### 15. No brand validation — empty brands pass silently
**File:** `lib/validator.ts`

Missing brands → poor catalog organization in Treez. Common across POS exports. Should be a warning.

### 16. `gel` subcategory keyword matches "gelato"
**File:** `lib/category-mapper.ts:145`

No word boundary on `/gel/i`. "Gelato" strain products classified as Topical → Gel if they end up in the Topical category path.

**Fix:** Use `\bgel\b` with word boundaries.

## MEDIUM — Architecture & Reliability

### 17. No URL validation on API proxy (security)
**File:** `entrypoints/background/index.ts:47, 97, 113`

`apiBaseUrl` comes from content script and gets the OAuth token attached. A bug or compromise in the content script could exfiltrate the token to an arbitrary URL.

**Fix:** Validate `apiBaseUrl` against known Treez API origins before attaching tokens.

### 18. Token transits through content script unnecessarily
**File:** `lib/messaging.ts:14`

Content scripts fetch the token and pass it back in messages. The background script should manage tokens internally.

### 19. No timeouts on fetch calls
**File:** `entrypoints/background/index.ts` (all `fetch()` calls)

No `AbortController`. A hanging API call blocks indefinitely until service worker termination.

### 20. Service worker restart loses `refreshInProgress` guard
**File:** `entrypoints/background/auth.ts:106`

In-memory flag. MV3 service worker restarts → two concurrent refreshes can fire.

### 21. `fullJoin` drops duplicate right-only rows
**File:** `lib/inventory-etl-helpers.ts:144-157`

Only first right-only row per key emitted. Doesn't match true full outer join semantics.

### 22. `arrayToCSV` doesn't handle `\r`
**File:** `lib/csv-generator.ts:51`

Catalog CSV generator misses carriage returns. Inventory generator handles them. Inconsistency.

### 23. No BOM in CSV output
**File:** `lib/csv-generator.ts`, `lib/inventory-csv-generator.ts`

Non-ASCII characters (accented brand names, em dashes) render as mojibake in Excel without BOM.

### 24. Floating point accumulation in `sumByGroup`
**File:** `lib/inventory-etl-helpers.ts:43-49`

Repeated `+=` on monetary values. Could produce `1234.5600000000002`. Mitigated by downstream `toFixed(2)` in some paths but not all.

## MEDIUM — UX & Accessibility

### 25. No ARIA roles or focus trap on drawer
**File:** `entrypoints/wizard-drawer.content/DrawerApp.tsx`

Drawer has no `role="dialog"`, `aria-modal`, `aria-labelledby`, or focus trap. Keyboard users can tab behind the backdrop. WCAG 2.1 AA requirement.

### 26. Inventory wizard has no "Done" button
**File:** `components/wizard/WizardShell.tsx:534-553`

`InventoryImportStep` has no `onDone` callback. After inventory import completes, no footer action — user must use X button.

### 27. Empty file flows through to import
A file with 0 data rows passes through all steps: "Transforming 0 rows..." → "All 0 rows valid" → import attempt. Should block at upload.

### 28. Mapping dropdowns lack `<label>` elements
**File:** `components/mapping/MappingRow.tsx:46-59`

Screen readers announce unlabeled dropdowns.

### 29. Inconsistent naming: "IMS Login" vs "Portal" vs "CS Tool"
Non-technical dispensary staff won't know what these mean. Use one user-friendly name.
