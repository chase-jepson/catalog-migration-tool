Create the ReviewStep component tree. Reference v1's ReviewStep.tsx (605 lines) for patterns but build fresh for v2's side panel UX.

**components/review/TransformPreview.tsx:**
- Table showing before/after columns for key transformed fields (category, subCategory, classification, weight/amount, uom)
- "Original" column shows raw source value, "Treez Value" column shows derived value
- Show first 20 rows with a "Show all N rows" toggle
- Compact layout for 400px side panel width

**components/review/ErrorBatchRow.tsx:**
- Props: `{ group: ErrorGroup; onFix: (rowIndices: number[], field: string, newValue: string) => void }`
- For fixType 'dropdown': render a `<select>` with dropdownOptions, "Apply to all N rows" button
- For fixType 'text': render a `<input type="text">`, "Apply to all N rows" button
- Show affected row count badge
- Per user decision: batch fix controls (dropdown of valid values for enum fields, text input for free-text)

**components/review/ErrorGroupList.tsx:**
- Props: `{ groups: ErrorGroup[]; onFix: (rowIndices: number[], field: string, newValue: string) => void }`
- Expandable groups: click to expand/collapse rows within each group
- Groups sorted by severity first (errors before warnings), then by row count descending
- Error count badge: red for errors, amber for warnings
- Format: "Category: 47 rows", "Weight: 12 rows" per user decision

**components/review/ReviewStep.tsx:**
- Props: `{ parsedFiles: ParsedFile[]; mappings: FieldMapping[]; onCanProceed: (can: boolean) => void; onDerivedRowsChange: (rows: DerivedRow[]) => void; onFixesChange: (fixes: RowFix[]) => void; fixes: RowFix[] }`
- On mount: call `deriveRows(mergedRows, mappings)` to produce DerivedRow[]
- Then call `applyFixes(derivedRows, fixes)` if fixes exist
- Then call `validateDerivedRows(fixedRows)` to produce ValidationResult
- Then call `groupErrors(validationResult.errors)` for display
- Show TransformPreview at top (collapsible)
- Show ErrorGroupList below
- "Re-validate" button: re-runs applyFixes + validateDerivedRows after user makes fixes
- onCanProceed(true) when errorCount === 0 (warnings are OK)
- When warnings exist, show count: "N warnings (non-blocking)" in a banner
- Loading spinner while deriveRows runs (can be <100ms but show spinner for UX consistency)

**Do NOT persist full DerivedRow[] to chrome.storage.local** -- per research anti-pattern, store only RowFix[] (fixes). Re-derive on restore from parsedFiles + mappings.
