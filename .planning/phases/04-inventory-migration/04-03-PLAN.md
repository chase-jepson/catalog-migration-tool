**Rewrite InventoryUploadStep.tsx** for multi-file upload with role assignment:

The upload step now needs to:
1. Allow uploading multiple files (up to 5)
2. For each uploaded file, show a card with a role dropdown (Inventory, Receipt Report, Vendors, Adjustments, Catalog Export)
3. Auto-detect role from file name or headers where possible:
   - File name contains "inventory" -> role: inventory
   - File name contains "receipt" -> role: receipts
   - File name contains "vendor" -> role: vendors
   - File name contains "adjustment" -> role: adjustments
   - File name contains "catalog" or "export" -> role: catalog_export
   - Headers contain "External Package ID" + "Receive Date" -> role: receipts
   - Headers contain "Vendor name" + "Vendor code" -> role: vendors
   - Headers contain "Product Key" + "Product Category" -> role: catalog_export
4. User can override auto-detected role via dropdown
5. Show which roles are assigned and which are missing
6. Inventory role is required (highlighted if missing); other roles are optional
7. Add a "Dispensary License" text input field below the file cards:
   - Label: "Dispensary License Number"
   - Placeholder: "e.g., C12-0000331-LIC"
   - Description text: "Required. This license number will be included in every row of the import."
   - Persisted in WizardShell state
8. POS detection still runs on all uploaded files (for column auto-mapping in next step)
9. canProceed requires: at least one file with role=inventory assigned, selectedPOS is set, selectedStore is not null, dispensaryLicense is non-empty
