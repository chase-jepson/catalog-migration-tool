  <name>Task 1: Background script with auth and side panel control</name>
  <files>entrypoints/background/index.ts, entrypoints/background/auth.ts, tests/auth.test.ts</files>
  <behavior>
    - decodeJwtPayload with valid JWT returns parsed payload object
    - decodeJwtPayload with malformed token returns null
    - isTokenExpired returns true when exp is in the past
    - isTokenExpired returns false when exp is in the future (beyond 60s buffer)
    - isTokenExpired returns true when exp is within 60s buffer
    - isTokenExpired returns true for malformed token
  </behavior>
  <action>
Write tests first, then implement.

1. Create `tests/auth.test.ts`:
   - Test `decodeJwtPayload` with valid base64-encoded JWT (create a test JWT with known payload)
   - Test `decodeJwtPayload` with malformed tokens (empty string, missing parts, invalid base64)
   - Test `isTokenExpired` with expired token (exp in past)
   - Test `isTokenExpired` with valid token (exp well in future)
   - Test `isTokenExpired` with token expiring within 60s buffer
   - Mock `Date.now` for deterministic expiry tests

2. Create `entrypoints/background/auth.ts`:
   - `StoredTokens` interface: { accessToken, refreshToken, expiresIn, idToken? }
   - `decodeJwtPayload(token: string): Record<string, unknown> | null` -- base64 decode the JWT payload section. Split on '.', verify 3 parts, atob the middle part with URL-safe base64 handling, JSON.parse. Return null on any error.
   - `isTokenExpired(token: string): boolean` -- decode payload, check `exp` claim, return true if current time > exp - 60 (60 second buffer). Return true if decode fails.
   - `getTokensViaScripting(tabId: number): Promise<StoredTokens | null>` -- use `chrome.scripting.executeScript` to read `localStorage.getItem('tz-tokens')` from the page. Parse JSON and return.
   - `refreshAccessToken(refreshToken: string, env: TreezEnv): Promise<StoredTokens | null>` -- POST to oauth endpoint (use `getApiBaseUrl` from lib/env.ts to derive oauth URL). For now, implement the structure but note the exact OAuth endpoint will need verification in Phase 2+.
   - `getValidToken(tabId: number, tabUrl: string): Promise<string | null>` -- get tokens via scripting, check if expired, refresh if needed. Include a refresh-in-progress guard (module-level promise variable) to prevent concurrent refreshes per Pitfall 4 from RESEARCH.md.
   - Export `decodeJwtPayload` and `isTokenExpired` for testing. Export `getValidToken` for the background script.

3. Create `entrypoints/background/index.ts`:
   - Use `defineBackground` from WXT
   - Set `browser.sidePanel.setPanelBehavior({ openPanelOnActionClick: true })`
   - Listen on `chrome.tabs.onUpdated` -- when tab URL changes, check if it matches import page patterns (use IMPORT_PAGE_PATTERNS from constants). Call `chrome.sidePanel.setOptions({ tabId, path: 'sidepanel.html', enabled: isImportPage })`.
   - ALSO listen on `chrome.webNavigation.onHistoryStateUpdated` for SPA navigation detection (per Pitfall 1 from RESEARCH.md). Same logic: check URL, enable/disable side panel.
   - Register `onMessage('getAuthToken')` handler -- calls `getValidToken` and returns the token.
   - Register `onMessage('openSidePanel')` handler -- calls `chrome.sidePanel.open({ tabId })` and stores the wizardType in `chrome.storage.session` so the side panel can read which wizard to show.
   - Add `webNavigation` to permissions in wxt.config.ts if not already present.
  </action>

  </verify>
