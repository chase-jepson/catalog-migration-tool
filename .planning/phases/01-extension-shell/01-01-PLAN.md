1. Run `pnpm dlx wxt@latest init . --template react` to scaffold. If this fails due to non-empty directory, manually create the required files.

2. Install dependencies:
```bash
pnpm i react react-dom
pnpm i -D @wxt-dev/module-react tailwindcss @tailwindcss/vite @webext-core/messaging typescript
```

3. Create `wxt.config.ts` exactly as specified in RESEARCH.md Pattern 1 -- complete WXT config with:
   - `extensionApi: 'chrome'` (Chrome-only, no browser polyfill)
   - `modules: ['@wxt-dev/module-react']`
   - `manifest` with name "Catalog Migration Tool", action: {}, permissions: ['storage', 'activeTab', 'tabs', 'sidePanel', 'scripting'], and all host_permissions for Treez domains (app, api, oauth, S3)
   - `hooks.build:manifestGenerated` to delete `side_panel` entry (controls it programmatically)
   - `vite` config with `@tailwindcss/vite` plugin

4. Create minimal sidepanel entrypoint files so WXT recognizes the sidepanel entry:
   - `entrypoints/sidepanel/index.html` -- basic HTML with div#root and script tag
   - `entrypoints/sidepanel/style.css` -- `@import "tailwindcss";`

5. Generate simple placeholder icons (16x16, 48x48, 128x128) in `assets/` or `public/icon/`. These are functional placeholders -- a colored square with "CM" text is fine. Use a simple SVG-to-PNG approach or just create minimal valid PNGs.

6. Verify the project builds: `pnpm wxt build`

    - detectEnvironment("https://app.sandbox.treez.io/treez-admin/import/home") returns "sandbox"
    - detectEnvironment("https://app.dev.treez.io/treez-admin/import/home") returns "dev"
    - detectEnvironment("https://example.com") returns null
    - detectEnvironment("invalid") returns null
    - getApiBaseUrl("production") returns "https://api.treez.io"

    - getApiBaseUrl("dev") returns "https://api-dev.treez.io"
  </behavior>
  <action>
1. Install test dependencies: `pnpm i -D vitest @webext-core/fake-browser jsdom`

2. Create `vitest.config.ts`:
   - Use `defineConfig` from vitest/config
   - Set environment to 'jsdom'
   - Set include pattern for `tests/**/*.test.ts`

3. Create `lib/env.ts` with:
   - `TreezEnv` type: 'production' | 'sandbox' | 'dev'
   - `ENV_MAP` record mapping hostnames to environments
   - `detectEnvironment(url: string): TreezEnv | null` -- extract hostname, look up in ENV_MAP
   - `getApiBaseUrl(env: TreezEnv): string` -- return correct API base URL per environment
   - Follow the exact pattern from RESEARCH.md "Environment Detection" code example

4. Create `lib/constants.ts` with:
   - `IMPORT_PAGE_PATTERNS`: array of URL match patterns for all three environments (`https://app.treez.io/treez-admin/import/home*`, sandbox, dev equivalents)
   - `STEP_LABELS`: `['Upload', 'Map', 'Review', 'Import']`
   - `TREEZ_HOSTS`: object mapping environment to app hostname
   - `IMPORT_PATH`: `/treez-admin/import/home`

5. Create `lib/messaging.ts` with:
   - `ProtocolMap` interface defining: `getAuthToken(data: { appUrl: string }): { token: string | null }` and `openSidePanel(data: { tabId: number; wizardType: 'catalog' | 'inventory' }): void`
   - Export `sendMessage` and `onMessage` from `defineExtensionMessaging<ProtocolMap>()`

6. Create `tests/env.test.ts` with tests for all behaviors listed above. Write tests FIRST (RED), then implement lib/env.ts (GREEN).
